# Система управления задачами с приоритетами

Консольная программа на Python для управления задачами. Проект разработан для проектной работы, демонстрирует применение базовых алгоритмов и структур данных: очередей, стеков и бинарных деревьев поиска (BST).

## Что умеет программа

* Добавлять задачи с названием, приоритетом, временем выполнения и дедлайном.
* Удалять и изменять задачи (с функцией пропуска параметров*).
* Отменять последние действия (добавление, удаление, изменение).
* Ставить задачи в очередь на выполнение и извлекать их по порядку.
* Выводить все задачи, отсортированные по дедлайну (через симметричный обход дерева).
* Быстро находить задачи с самым ранним и самым поздним дедлайном.
* Фильтровать список задач по уровню приоритета.*
* Автоматически сохранять и загружать все данные (включая очередь) в файл `tasks.json`.*
* Защищать программу от некорректного ввода (строгая валидация дат и чисел).*

  «*» - это дополнительный функционал

## Структуры данных

| Структура | Где используется | Реализация |
| :--- | :--- | :--- |
| **Стек (LIFO)** | История действий для отмены (Undo) | На базе стандартного списка Python (`append` / `pop`) |
| **Очередь (FIFO)** | Очередь задач на выполнение | На базе стандартного списка Python (`pop(0)`) |
| **Бинарное дерево (BST)** | Сортировка и быстрый поиск по дедлайну | Собственные классы `BSTNode` и `BinarySearchTree` |

## Как запустить программу

Необходимо клонировать репозиторий:
```bash
git clone https://github.com/Vhs167/Task.git

```

Запустить программу (Python 3.8+):

```bash
python main.py

```

## Как пользоваться программой

После запуска откроется главное меню:

```text
=======================================================
      СИСТЕМА УПРАВЛЕНИЯ ЗАДАЧАМИ С ПРИОРИТЕТАМИ
=======================================================
1. Добавить новую задачу
2. Удалить задачу по названию
3. Изменить параметры задачи
4. Отменить последнее действие
5. Добавить задачу в Очередь на исполнение
6. Выполнить следующую задачу из очереди
7. Показать задачи по возрастанию дедлайна
8. Быстрый поиск в дереве: Самый ранние и поздние сроки
9. Фильтрация задач по приоритету
10. Сохранить все изменения в файл
0. Выход из программы
=======================================================
Выберите действие: 
```
## Примеры сессиий
Добавление новой задачи
```text
Выберите действие: 1
Название задачи: Подготовить презентацию
Приоритет (1 - высокий, 2 - средний, 3 - низкий): 1
Время выполнения (в часах): 3
Дедлайн (в формате ДД-ММ-ГГГГ): 18-06-2026
Задача успешно добавлена: Подготовить презентацию
```
Изменение параметров задачи
```text
Выберите действие: 3
Введите название задачи для изменения: Подготовить презентацию
Новый приоритет (текущий: 1) [Enter для сохранения]: 
Новое время выполнения (текущее: 3ч) [Enter для сохранения]: 5
Новый дедлайн (текущий: 18-06-2026) [Enter для сохранения]: 

Приоритет оставлен прежним: 1
Время выполнения оставлено прежним: 5ч
Дедлайн оставлен прежним: 18-06-2026
Задача «Подготовить презентацию» успешно изменена.
```
Добавление задачи в очередь на исполнение
```text
Выберите действие: 5
Введите название задачи для постановки в очередь: Подготовить презентацию
Задача «Подготовить презентацию» встала в очередь на исполнение.
```
Выполнение задачи из очереди
```text
Выберите действие: 6
Выполняется задача: «Подготовить презентацию» | Приоритет: 1 | Время: 5ч | Дедлайн: 18-06-2026
```
Выход из программы (с сохранением)
```text
Выберите действие: 0
Данные сохранены в файл tasks.json.
Программа завершена. До встречи!
```

## Структура проекта

| Файл | Описание |
| --- | --- |
| `models.py` | Базовый класс задачи, логика перевода форматов и дат |
| `structures.py` | Алгоритмы очереди, стека и бинарного дерева поиска |
| `manager.py` | Главный контроллер, объединяющий логику работы |
| `main.py` | Точка входа, безопасный ввод и консольный интерфейс |
| `tasks.json` | База данных программы (создается автоматически) |
